# Demo Recording Checklist

## Before recording

- Close unrelated windows and tabs
- Prepare the project folder in Finder or terminal
- Open the key files in advance
- Set browser/editor zoom so text is readable
- Silence notifications

## Files to prepare on screen

- `README.md`
- `docs/problem_statement.md`
- `outputs/reports/data_dictionary_inference.md`
- `outputs/tables/best_model_summary.csv`
- `outputs/figures/feature_importance_by_target.png`
- `outputs/figures/feature_selection_comparison.png`
- `outputs/figures/predicted_vs_actual_panels_final.png`
- `outputs/reports/final_comparison_report.md`

## Recording order

1. Project overview
2. Verified problem statement
3. Real-data schema verification
4. Feature sets and models
5. Best results table
6. Important figures
7. Final comparison with paper methodology
8. Close with reproducibility and saved models

## What to avoid saying

- Do not claim one universal best model for all toxins
- Do not say the session demo figure is strict validation evidence
- Do not present `Acid` as definitively confirmed uric acid without qualification
- Do not say feature selection improved every metric; it mostly improved interpretability

## Best closing sentence

This project reproduces the paper idea, extends it with fuller feature use and multi-model benchmarking, and turns it into a reproducible AI-assisted end-to-end workflow with transparent handling of methodological risks.
